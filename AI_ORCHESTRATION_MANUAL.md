# 🛡️ AI Orchestration Manual: Sovereign Engine v8.0

This manual codifies the operational protocols for the **Sovereign Engine**. Use this guide to navigate the Directorial interface and ensure a successful 400-page manuscript resurrection.

---

## 1. The Handshake Protocol (Connection)
Before any mission, you must establish a secure handshake with the AI providers.
- **Provider Cards**: Located in the **Vault (Settings)**.
- **Handshake Logic**: Enter your API Key and click **"Test Connection"**.
- **Portfolio Discovery**: A successful handshake will automatically populate the model list with that provider's active portfolio (e.g., Gemini 3.1 Pro, GPT-4o).

## 2. Mission Alignment (Task Assignment)
The workstation allows you to assign specific engines to specific "Mission Leads."
- **👁️ Transcription Lead**: The engine responsible for ocular scanning and OCR.
  - *Recommendation*: **Groq (Llama-3.2-11B)** for high-velocity batch scanning.
- **🏛️ Boardroom Lead**: The engine responsible for deep narrative analysis, logic, and chapter audits.
  - *Recommendation*: **Gemini 3.1 Pro** for massive context and logical depth.
- **✍️ Creative Prose**: The engine used for style-mirroring and creative drafting.
  - *Recommendation*: **Claude 3.5 Sonnet** for high-fidelity prose mirroring.

## 3. Directorial Guidance (⭐ Expert Choice)
The interface features **Expert Guidance Badges** to help you make optimal choices:
- **⭐ [BEST FOR TRANSCRIPTION]**: Indicates engines optimized for OCR and speed.
- **⭐ [BEST FOR LOGIC]**: Indicates engines optimized for complex reasoning.
- **Directorial Advisory**: Each card features a real-time advisory note explaining the specific strengths of that provider (e.g., Groq's LPU architecture).

## 4. Spectrum Failover (Resilience)
For long-running 400-page missions, you must configure a **Failover Gear**.
- **The Concept**: If your primary engine (e.g., Groq) hits a rate limit or a network blackout, the system will instantly shift to the Fallback engine.
- **Configuration**:
  1. Pick your primary (e.g., Groq + Llama 3.2).
  2. Pick your fallback on a DIFFERENT provider (e.g., Gemini + 1.5 Flash).
  3. Click **"Assign as Failover Fallback"** on the secondary card.

## 5. Visual Telemetry (The HUD)
While a mission is active, monitor the **Resurrection Dashboard**:
- **Active Engine Badge**: Shows exactly which engine is processing the current page.
- **Gear Shift Indicator**: If a failover occurs, the badge will update to reflect the new active gear.
